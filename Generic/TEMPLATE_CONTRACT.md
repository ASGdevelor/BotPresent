# Контракт шаблона презентации

`Generic/index.html` — базовая бизнес-презентация без продажи AI-блогеров. `Generic/index3.html` — отдельный продающий шаблон AI-блогеров; он выбирается автоматически после ответа пользователя «да». Генератор не собирает слайды заново: он читает выбранный файл как UTF-8 и заменяет маркеры `{{UPPER_SNAKE_CASE}}` значениями, полученными с сайта, из открытых источников и из настроек пользователя. Если во внешнем каталоге шаблонов нет `index3.html`, генератор безопасно возвращается к `index.html`.

## Что нельзя менять без обновления генератора

- Восемь элементов `<section data-page="1">` … `<section data-page="8">` задают восемь смысловых блоков HTML-презентации. PDF сохраняет их непрерывное расположение на одном листе с теми же размерами и отступами, что у десктопной HTML-версии шириной 1280 px.
- Маркеры `MARKET_CHART_SVG`, `CHANNEL_CHART_SVG`, `ECONOMY_CHART_SVG`, `VOLUME_CHART_SVG`, `TRUST_CHART_SVG`, `REACH_CHART_SVG`, `ROADMAP_CHART_SVG` вставляют автономные векторные диаграммы.
- В обоих шаблонах каждому SVG соответствует интерактивный Chart.js canvas и JSON-маркер `*_CHART_DATA`. Закреплённый Chart.js 4.5.1 встраивается в итоговый HTML вместо `<!-- BOT_PRESENT_CHART_JS -->`; SVG остаётся печатным и no-JS fallback.
- В режиме AI-блогеров три полноцветных talking-head GIF выбираются для компании и тем её направлений. Они встроены как автономные `data:` URI; локальные фотографии и каталог `Generic/assets` не используются.
- Локальные правки разделов хранятся в `preferences.sectionEdits`: заголовок и основной текст заменяют только соответствующие маркеры раздела, а дополнительное фото выводится через `PAGE_*_EDIT` и одинаково попадает в HTML и PDF.
- Все маркеры должны иметь формат `{{NAME}}`. Неизвестный или незаполненный маркер останавливает генерацию с понятной ошибкой.
- PDF создаётся через Chrome DevTools Protocol в экранном media-режиме: фактические ширина и высота HTML передаются как размер единственного листа, поля равны нулю, фон печатается без цветовой коррекции.

## Источники данных

| Группа | Маркеры | Источник |
|---|---|---|
| Компания | `COMPANY`, `HEADLINE`, `DESCRIPTION`, `WEBSITE`, `ABOUT_*`, `PRODUCT_*` | сведения о бизнесе из HTML, meta, JSON-LD и публичных разделов |
| Логотип | `LOGO_URL` | JSON-LD `logo`, logo-img, header-img, затем Open Graph/favicon; URL приводится к абсолютному HTTP(S) |
| Палитра | `PRIMARY_COLOR`, `SECONDARY_COLOR`, `THEME_*` | выбранная пользователем тема; для темы 1 — фирменные цвета сайта |
| Диагностика | `KPI_*`, `ABOUT_*` | оценка ясности оффера, доверия, пути к обращению и контентного потенциала |
| Рынок и аудитория | `MARKET_*`, `CHANNEL_*` | сопоставимые актуальные показатели и сегментационная модель категории |
| Конкуренты | `COMPETITOR_CARDS`, `TRUST_*`, `REACH_*` | результаты поисковой выдачи и факторы выбора в категории |
| Графики | `*_CHART_DATA`, `*_CHART_SVG` | Chart.js для HTML и самостоятельно созданные SVG для PDF/no-JS; оценочная методика описывается в подписи без математических префиксов у значений |
| AI-блогеры | `VIDEO_*_MEDIA`, `VIDEO_*` | Полноцветный AI-video GIF выбирается сервером по роли и теме компании, без CDN |
| Контакты | `CONTACT_*` | `mailto:`, `tel:` и публичные Telegram-ссылки сайта |

Если проверяемого рыночного числа нет, презентация использует явно оценочную бизнес-модель — силу бренда, структуру аудитории или приоритет запуска. Нулевые заглушки, количество проанализированных страниц и сообщения «данные не найдены» на слайды не выводятся.

## Как точнее описывать новые поля

Для расширения шаблона добавляйте рядом с новым маркером HTML-комментарий следующего вида:

```html
<!-- bind: TEAM_SIZE; source: official-site; type: integer; required: false; fallback: "не опубликовано" -->
<strong>{{TEAM_SIZE}}</strong>
```

Для графика укажите единицу, допустимое число точек и требование к источнику:

```html
<!-- bind: GROWTH_CHART_SVG; source: external-public-or-analysis; unit: percent; points: 2..6; visible-citation: false -->
<div class="cv">{{GROWTH_CHART_SVG}}</div>
```

После добавления маркера нужно добавить его значение в `renderPresentationTemplate`, источник в `WebsiteFacts` и регрессионный тест. Это даёт более точный результат, не меняя визуальную структуру шаблона.
