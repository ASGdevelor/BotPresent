# Контракт шаблона презентации

`Generic/index.html` — единственный источник структуры презентации. Генератор не собирает слайды заново: он читает этот файл как UTF-8 и заменяет маркеры `{{UPPER_SNAKE_CASE}}` значениями, полученными с сайта, из открытых источников и из настроек пользователя.

## Что нельзя менять без обновления генератора

- Восемь элементов `<section data-page="1">` … `<section data-page="8">` задают восемь HTML/PDF-слайдов.
- Идентификаторы `cMarket`, `cChannel`, `cEconomy`, `cVolume`, `cTrust`, `cReach`, `cRoadmap` связывают canvas с данными графиков.
- Три элемента `<video>` и их исходные URL являются частью исходного дизайна.
- Все маркеры должны иметь формат `{{NAME}}`. Неизвестный или незаполненный маркер останавливает генерацию с понятной ошибкой.
- Печатный блок `@media print` фиксирует один `section` на один слайд PDF 16:9.

## Источники данных

| Группа | Маркеры | Источник |
|---|---|---|
| Компания | `COMPANY`, `DESCRIPTION`, `WEBSITE`, `ABOUT_*`, `PRODUCT_*` | HTML, meta, JSON-LD и публичные разделы сайта пользователя |
| Логотип | `LOGO_URL` | JSON-LD `logo`, logo-img, header-img, затем Open Graph/favicon; URL приводится к абсолютному HTTP(S) |
| Палитра | `PRIMARY_COLOR`, `SECONDARY_COLOR`, `THEME_*` | выбранная пользователем тема; для темы 1 — фирменные цвета сайта |
| Графики | `*_CHART_DATA`, `BENCH_*`, `MARKET_SOURCE` | числовые факты с найденной страницы и точная ссылка на неё |
| Контакты | `CONTACT_*` | `mailto:`, `tel:` и публичные Telegram-ссылки сайта |

Если проверяемого числа нет, шаблон показывает нулевой fallback и текст «данные не найдены». Генератор не должен придумывать рынок, проценты или финансовые показатели.

## Как точнее описывать новые поля

Для расширения шаблона добавляйте рядом с новым маркером HTML-комментарий следующего вида:

```html
<!-- bind: TEAM_SIZE; source: official-site; type: integer; required: false; fallback: "не опубликовано" -->
<strong>{{TEAM_SIZE}}</strong>
```

Для графика укажите единицу, допустимое число точек и требование к источнику:

```html
<!-- bind: GROWTH_CHART_DATA; source: external-public; unit: percent; points: 2..6; citation: required -->
<canvas id="cGrowth"></canvas>
```

После добавления маркера нужно добавить его значение в `renderPresentationTemplate`, источник в `WebsiteFacts` и регрессионный тест. Это даёт более точный результат, не меняя визуальную структуру шаблона.
