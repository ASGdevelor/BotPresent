# Устройство лидогенерации

## Назначение

Модуль ищет потенциальных B2B-клиентов по анкете пользователя и возвращает список успешно проанализированных компаний. Компании с публичным Telegram-контактом располагаются первыми, но сайты без контакта также сохраняются. Модуль не гарантирует, что владелец контакта принимает коммерческие предложения.

## Модель данных

Анкета типизирована интерфейсом `LeadCriteria`:

| Поле | Смысл |
| --- | --- |
| `whoCanBuy` | Целевая аудитория и её потребность |
| `whoToFind` | Тип искомых компаний |
| `whereToSearch` | География, каталог, домены или URL |
| `offer` | Предлагаемая услуга и польза |
| `exclusions` | Нежелательные типы компаний |

Результат `LeadGenerationResult` содержит исходные критерии, массив `CompanyLead`, количество проанализированных сайтов и предупреждения. `CompanyLead` включает `siteName` и полный URL. `TelegramContact` содержит handle, предполагаемую роль, подпись и URL страницы-источника.

Роль имеет union-тип `ContactRole`: `director`, `sales`, `manager`, `employee`, `company` или `unknown`.

## Состояния диалога

В `bot.ts` используется дискриминируемое объединение:

- `PresentationSession` — ожидание темы презентации;
- `LeadGenerationSession` — строго типизированное текущее поле `LeadField` и частично заполненный `LeadCriteria`.

После каждого ответа состояние обновляется в памяти. Перед запуском поиска применяется type guard `isCompleteLeadCriteria`. Команды `/start` и `/cancel` удаляют состояние пользователя.

## Выбор сайтов

Поле «Где искать» обрабатывается двумя способами:

1. Если найдены URL или домены, анализируются именно они.
2. Если URL нет, формируется запрос к HTML-выдаче Bing из полей `whoToFind`, `whoCanBuy`, `whereToSearch` и слов «контакты telegram». Переходные ссылки Bing декодируются до исходных сайтов.

Количество кандидатов ограничено десятью. Социальные сети, Telegram и поисковые домены исключаются из кандидатов, так как задача модуля — анализировать сайт компании и связать контакт с компанией.

## Анализ компании

Для каждого кандидата модуль:

1. Загружает главную или указанную страницу.
2. Извлекает название из Open Graph, метаданных, `h1` или `title`.
3. Извлекает описание из meta description либо видимого текста.
4. Проверяет фразы из поля исключений.
5. Находит до двух внутренних страниц с признаками «контакты» или «о компании».
6. Собирает ссылки `t.me/<handle>` и подписи контактов.
7. Удаляет дубликаты, invite-ссылки и handles, заканчивающиеся на `bot`.
8. Определяет роль по подписи и окружающему открытому тексту.
9. Сортирует контакты: руководство, продажи, менеджеры, сотрудники, неизвестные, корпоративные.
10. Включает компанию в отчёт даже при отсутствии Telegram-контакта.

Сайты анализируются параллельно через `Promise.allSettled`. Ошибка одного кандидата добавляется в `warnings` и не прерывает остальные запросы.

## Сетевая безопасность

`public-web.ts` выполняет проверки до каждого запроса:

- разрешает только `http:` и `https:`;
- удаляет логин и пароль из URL;
- запрещает localhost, `.local`, loopback, private, link-local и служебные диапазоны;
- разрешает DNS и повторно проверяет полученные IP-адреса;
- использует ручные редиректы и проверяет каждый новый URL;
- ограничивает число редиректов тремя;
- устанавливает тайм-аут 12 секунд;
- принимает только HTML и ограничивает документ двумя мегабайтами.

Эти проверки снижают риск SSRF и случайного обращения к внутренним сервисам.

## Обработка ошибок

- `PublicWebError` описывает ошибки URL, DNS, HTTP, типа документа и размера.
- `LeadGenerationError` содержит безопасное сообщение для пользователя при отказе поиска.
- Необязательные контактные страницы обёрнуты в локальный `try/catch`.
- Анализ всех кандидатов использует `Promise.allSettled`.
- Обработчик Telegram оборачивает генерацию отчёта в `try/catch` и не показывает пользователю внутренний stack trace.
- Общий `bot.catch` разделяет ошибки Telegram API, HTTP и неожиданные исключения.

## Ограничения

- HTML-поиск без API-ключа зависит от доступности и разметки Bing.
- JavaScript-only сайты могут не содержать контакты в исходном HTML.
- Бот не обходит CAPTCHA и авторизацию.
- Текстовая фильтрация исключений является эвристической.
- Публичный Telegram handle не обязательно принадлежит конкретному сотруднику отдела продаж.
- Роль контакта определяется эвристически и может быть ошибочной; источник сохраняется для ручной проверки.
- Для стабильного коммерческого использования рекомендуется подключить официальный Search API и CRM с правилами хранения и удаления данных.
