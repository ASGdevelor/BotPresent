# Журнал изменений

## Не выпущено

### Добавлено

- Создание HTML- и PDF-презентаций по публичному сайту с отображением прогресса.
- Локальное хранение, просмотр и повторная генерация презентаций по ID.
- Пакетное создание презентаций для результатов лидогенерации.
- Расширенный поиск до 60 сайтов и CSV-выгрузка контактов.
- Двухступенчатое интернет-исследование для графиков: основной и расширенный поисковые запросы.
- Резервное извлечение проверяемых чисел из поисковых сниппетов, когда исходная HTML-страница недоступна.
- Отдельные графики для показателей официального сайта и внешних отраслевых источников.
- Явная маркировка приблизительных чисел из поисковых сниппетов знаком `≈` и отдельный статус исследования `estimated`.
- Аналитические графики структуры сайта вместо пустых диаграмм при отсутствии отраслевой статистики.
- Встраивание изображений в автономный HTML перед печатью PDF.
- Выбор режима после ввода сайта: статичное предложение AI-блогеров или обычная бизнес-презентация.
- Эвристический анализ готовности бизнеса и контент-потенциала ниши с явно подписанной методикой приблизительных оценок.
- Автономные Chart.js-диаграммы в бизнес-шаблоне и шаблоне AI-блогеров с крупными подписями и SVG-fallback для PDF.
- Три тематических анимированных GIF AI-блогеров, генерируемых сервером в выбранной палитре при каждой новой презентации.
- Региональный блок запуска AI-блогеров для России, Беларуси, Казахстана и других рынков СНГ.

### Исправлено

- PDF снова создаётся одним непрерывным листом через Chrome DevTools Protocol и повторяет экранную HTML-вёрстку при фиксированной ширине 1280 px.
- С диаграмм удалены постоянно отображаемые числовые подписи значений; точные значения остаются доступны только в интерактивных подсказках HTML.
- Проценты роста больше не смешиваются с долями рынка только из-за одинаковой единицы измерения.
- Годы в поисковых запросах и оценке актуальности теперь вычисляются динамически.
- Тест разбора данных графиков работает с переводами строк Windows и Unix.
- С финального слайда удалён общий список источников; ссылки остаются у соответствующих графиков и в метаданных презентации.
- Удалены жёстко заданные контакты, чужие видео и учебные формулировки, не связанные с анализируемой компанией.
- Служебные JSON-фрагменты отфильтровываются из заголовков и направлений презентации.
- В PDF закреплена двухколоночная сетка первого слайда и добавлены автономные кадры трёх тематических AI-GIF.
- Исправлено растягивание Chart.js canvas из-за конфликтующей процентной высоты; диаграммы больше не превращаются в длинные пустые области.
- Удалён локальный roster AI-блогеров и зависимость презентаций от фото из `Generic/assets`.
- Главный подзаголовок теперь собирается как персональный вывод: что изучено, в чём сила и слабость бренда и какое решение предлагается.
- В шаге 6 устаревающий статичный блок цифр заменён двумя диаграммами в равных колонках.
- Значения оценочных диаграмм больше не получают знак `≈` и длинную расшифровку процента; методика остаётся в подписи графика.
- Каждый сектор и столбец получает отдельный оттенок палитры, выбранной пользователем до генерации.

### Документация

- README, пользовательские сценарии и инструкция развёртывания приведены в соответствие с текущим поведением бота.

## 1.2.0 — 2026-07-10

### Добавлено

- Предполагаемые роли Telegram-контактов: руководство, продажи, менеджер, сотрудник, корпоративный или неизвестный.
- URL страницы-источника для ручной проверки контакта.
- Постоянная история сообщений в SQLite.
- Команды `/history` и `/clear_history`.
- Ограничение истории: 200 записей на пользователя в чате и 90 дней.
- Docker volume `./data` для сохранения истории между перезапусками.

### Изменено

- Компании без найденного Telegram-контакта теперь остаются в отчёте.
- Компании и контакты с наиболее полезными ролями выводятся первыми.
- Версия пакета повышена до `1.2.0`.

### Конфиденциальность

- Бинарные документы и токен не сохраняются в истории.
- Пользователь может выгрузить только историю личного чата и удалить свои записи.
- Файлы SQLite исключены из Git.

## 1.1.0 — 2026-07-10

### Изменено

- Кнопка **«Провести исследование»** заменена на **«Лидогенерация»**.
- Одношаговый запрос заменён типизированной анкетой из пяти вопросов.
- README и пользовательская документация обновлены под новый сценарий.
- Описание пакета обновлено, версия повышена до `1.1.0`.

### Добавлено

- Поиск сайтов через публичную HTML-выдачу Bing или анализ явно заданных URL.
- Извлечение названия, описания, контактных страниц и публичных Telegram handles.
- Markdown-отчёт со списком компаний и контактами.
- Типы `LeadCriteria`, `CompanyLead`, `TelegramContact` и `LeadGenerationResult`.
- Частичная устойчивость: недоступность одного сайта не отменяет анализ остальных.
- Проверки HTTP-статусов, типа и размера ответа, DNS, редиректов и тайм-аутов.
- Unit-тесты парсинга лидов и сетевой защиты.

### Безопасность

- Запрещены запросы к localhost, приватным и служебным IP-адресам.
- Не сохраняются Telegram invite-ссылки и аккаунты ботов.
- Собираются только контакты, опубликованные на открытых страницах.

### Удалено

- Интеграция с Wikipedia и файл `src/services/research.ts`.
