# Действия в BotPresent

Документ описывает все пользовательские действия, ответы бота и получаемые файлы.

## Начало работы

1. Откройте [@Roky_robotPresentationbot](https://t.me/Roky_robotPresentationbot) в Telegram.
2. Нажмите **Запустить** или отправьте команду `/start`.
3. Бот покажет главное меню с двумя кнопками:
   - **📊 Создать презентацию**;
   - **🔎 Провести исследование**.

Если меню пропало, отправьте `/menu`. Главное меню также появляется после завершения или отмены действия.

## Создание презентации

### Последовательность действий

1. Нажмите **📊 Создать презентацию**.
2. Дождитесь сообщения с просьбой указать тему.
3. Отправьте тему одним текстовым сообщением, например:

   ```text
   Применение искусственного интеллекта в школьном образовании
   ```

4. Бот ответит «Готовлю презентацию…».
5. После обработки бот отправит документ PowerPoint в формате `.pptx`.

### Содержимое презентации

Презентация состоит из шести редактируемых слайдов:

1. Титульный слайд.
2. Контекст и актуальность.
3. Ключевые аспекты.
4. Возможности и ограничения.
5. План действий.
6. Выводы.

Файл является базовой структурой. Перед выступлением рекомендуется добавить проверенные факты, изображения, диаграммы и адаптировать формулировки под аудиторию.

### Результат

- Формат: Microsoft PowerPoint `.pptx`.
- Имя файла формируется из темы.
- Файл можно открыть в PowerPoint, LibreOffice Impress, Keynote или импортировать в Google Slides.
- Временная серверная копия удаляется сразу после отправки в Telegram.

## Проведение исследования

### Последовательность действий

1. Нажмите **🔎 Провести исследование**.
2. Дождитесь сообщения с просьбой указать тему.
3. Отправьте тему одним текстовым сообщением, например:

   ```text
   Возобновляемая энергетика в России
   ```

4. Бот ответит «Ищу и собираю материалы…».
5. После обработки бот отправит текстовый документ в формате Markdown `.md`.

### Содержимое исследования

Отчёт включает:

- название и время формирования;
- до пяти связанных материалов русскоязычной Wikipedia;
- краткое описание каждого найденного материала;
- прямую ссылку на каждый использованный материал;
- рекомендуемые следующие шаги для углубления исследования.

### Результат и ограничения

- Формат: Markdown `.md`, кодировка UTF-8.
- Отчёт создаётся в памяти и не сохраняется на сервере.
- Для работы требуется доступ сервера к `ru.wikipedia.org`.
- Wikipedia используется как стартовая справочная точка, а не как первичный источник.
- Важные факты следует сверять с актуальными исследованиями, официальными документами и первичными публикациями.

## Команды управления

| Команда | Когда использовать | Результат |
| --- | --- | --- |
| `/start` | Первый запуск или полный сброс сценария | Сбрасывает ожидаемое действие и показывает главное меню |
| `/menu` | Нужно снова увидеть кнопки | Показывает главное меню |
| `/cancel` | Бот ожидает тему, но действие выполнять не нужно | Отменяет ожидание темы и возвращает кнопки |

## Правила ввода темы

- Отправляйте тему текстом после выбора кнопки.
- Минимальная длина — 3 символа.
- Максимальная обрабатываемая длина — 160 символов; более длинный текст будет сокращён.
- Лишние пробелы и переводы строк автоматически убираются.
- Чем конкретнее тема, тем релевантнее будет результат исследования.

Хороший запрос:

```text
Преимущества и риски удалённой работы для IT-команд
```

Слишком общий запрос:

```text
Работа
```

## Состояния и ответы бота

| Действие или состояние | Ответ бота |
| --- | --- |
| Получена команда `/start` | Приветствие и две кнопки |
| Выбрано создание презентации | Запрос темы презентации |
| Выбрано исследование | Запрос темы исследования |
| Тема короче 3 символов | Просьба отправить более длинную тему |
| Текст отправлен без выбора кнопки | Просьба сначала выбрать действие |
| Идёт создание презентации | Сообщение о подготовке документа |
| Идёт исследование | Сообщение о поиске материалов |
| Действие завершено | Файл и главное меню |
| Выполнена команда `/cancel` | Подтверждение отмены и главное меню |
| Произошла ошибка | Сообщение с предложением повторить попытку |

## Повторное использование

После получения файла можно сразу выбрать следующую кнопку. Каждый новый запрос обрабатывается независимо. Если после выбора одного действия нажать другую кнопку, бот будет ожидать тему уже для последнего выбранного действия.

## Типовые проблемы

### Бот не отвечает

Убедитесь, что процесс запущен командой `bun run start`. При серверном развёртывании проверьте журнал systemd или Docker Compose.

### Не создаётся исследование

Проверьте исходящий HTTPS-доступ к Telegram API и `ru.wikipedia.org`. Попробуйте сформулировать тему конкретнее.

### Не открывается презентация

Скачайте документ полностью и откройте его приложением с поддержкой `.pptx`. Не переименовывайте расширение файла.

### Кнопки не отображаются

Отправьте `/menu` или `/start`. В настольном Telegram кнопки могут быть свёрнуты значком клавиатуры рядом с полем ввода.

## Сценарий для быстрой проверки

1. Отправьте `/start` и убедитесь, что появились две кнопки.
2. Создайте презентацию на тему «Кибербезопасность малого бизнеса» и проверьте полученный `.pptx`.
3. Проведите исследование на ту же тему и проверьте `.md` и ссылки.
4. Нажмите любую кнопку, затем отправьте `/cancel`.
5. Убедитесь, что бот подтвердил отмену и снова показал меню.
