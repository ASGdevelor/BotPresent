# Действия в BotPresent

Документ описывает пользовательские сценарии, ответы бота и получаемые файлы.

## Начало работы

1. Откройте [@Roky_robotPresentationbot](https://t.me/Roky_robotPresentationbot).
2. Нажмите **Запустить** или отправьте `/start`.
3. Выберите одну из двух кнопок:
   - **📊 Презентации**;
   - **🎯 Лидогенерация**.

Команда `/menu` повторно показывает кнопки. Команда `/cancel` отменяет текущий сценарий.

## Создание презентации

1. Нажмите **📊 Презентации**, затем **➕ Создать презентацию**.
2. Отправьте публичный HTTP- или HTTPS-адрес сайта компании.
3. Следите за сообщением о прогрессе от 0 до 100%.
4. Получите редактируемый `index.html` и PDF. Если Microsoft Edge или Google Chrome не найден, бот отправит HTML и предупредит, что PDF недоступен.

Пример темы:

```text
https://example.com
```

Бот собирает с главной и связанных страниц название, описание, направления работы и публичные контакты, а затем заполняет многоэкранный HTML-шаблон. Перед отправкой клиенту проверьте автоматически собранные факты.

### Просмотр и изменение презентаций

- **📁 Мои презентации** показывает сохранённые презентации и повторно отправляет файлы первых десяти.
- **✏️ Изменить презентацию** запрашивает ID. Отправьте только ID для повторной генерации с прежнего сайта или `ID новый-адрес-сайта` для замены источника.
- Файлы хранятся локально в `data/presentations/` и исключены из Git.

## Лидогенерация

### Шаг 1. Кому можно продать услуги

Опишите целевую аудиторию и потребность, которую решает услуга.

```text
Компаниям с отделом продаж от пяти человек, которые ведут отчётность вручную
```

### Шаг 2. Кого ищем

Укажите тип, отрасль или размер компаний.

```text
Региональные интеграторы CRM и агентства автоматизации
```

### Шаг 3. Где искать

Можно указать географию или конкретные публичные сайты.

```text
Москва и Московская область
```

Для наиболее предсказуемого результата перечислите домены или URL:

```text
example-company.ru, https://company.example/contacts
```

### Шаг 4. Что предложить

Кратко опишите услугу и ожидаемую пользу.

```text
Автоматизацию еженедельной отчётности и сокращение ручной работы менеджеров
```

### Шаг 5. Кого не брать

Перечислите исключения через запятую.

```text
Фрилансеры, государственные организации, компании без отдела продаж
```

Если исключений нет, отправьте `нет` или `-`.

### Получение результата

После пятого ответа бот:

1. Формирует поисковый запрос или использует явно указанные сайты.
2. Проверяет до 60 публичных сайтов.
3. Анализирует главную страницу, услуги, каталог, руководство и контакты.
4. Применяет указанные исключения.
5. Определяет предполагаемую роль каждого открытого Telegram-контакта.
6. Сортирует компании по релевантности, а при равной оценке ставит сайты с контактами первыми.
7. Отправляет отчёт `leads-<тип-компаний>.csv`.

Для каждой найденной компании отчёт содержит:

- название;
- сайт;
- регион, тип совпадения и оценку релевантности;
- найденных представителей компании и их предполагаемые роли;
- публичные Telegram-контакты, телефоны, email и ссылки на социальные сети.

Роли: директор/владелец, продажи/развитие бизнеса, менеджер, сотрудник, корпоративный контакт или «роль не указана». Классификация основана на подписи и окружающем тексте, поэтому перед обращением её необходимо проверить вручную. После выгрузки доступна кнопка **🗂 Презентации для всех лидов**, которая последовательно создаёт HTML/PDF для найденных сайтов.

## Команды

| Команда | Действие |
| --- | --- |
| `/start` | Сбросить текущий сценарий и показать главное меню |
| `/menu` | Показать главное меню без сброса заполнения |
| `/cancel` | Отменить текущее действие |
| `/history` | Выгрузить последние 100 сообщений личного чата в `.md` |
| `/clear_history` | Удалить всю свою историю текущего чата |

## Проверки ввода

- Ответ на каждый вопрос должен содержать не менее 3 символов.
- Ответ анкеты нормализуется и ограничивается 500 символами.
- Исключения можно пропустить ответом `нет` или `-`.
- Нажатие другой основной кнопки начинает новый сценарий и заменяет незавершённый.
- При некорректном внутреннем состоянии анкета безопасно сбрасывается.

## Ошибки

| Ситуация | Поведение |
| --- | --- |
| Текст отправлен до выбора кнопки | Бот просит выбрать действие |
| Сайт презентации недоступен или адрес неверен | Бот сообщает об ошибке и оставляет меню презентаций |
| Браузер для печати не найден | HTML отправляется без PDF с пояснением |
| Ответ слишком короткий | Бот повторно ожидает текущий ответ |
| Поиск не вернул сайты | Бот предлагает уточнить критерии или указать домены |
| Один сайт недоступен | Остальные сайты продолжают анализироваться |
| Не найден Telegram | Компания остаётся в отчёте с отметкой «не найдены» |
| Общий сетевой сбой | Бот возвращает понятное сообщение и главное меню |
| Выполнена команда `/cancel` | Анкета удаляется из памяти, появляется меню |

## Безопасность и допустимое использование

- Обрабатываются только публичные HTTP- и HTTPS-страницы.
- Доступ к `localhost`, локальной сети и служебным IP-адресам заблокирован.
- Проверяются адреса каждого редиректа.
- Приватные Telegram invite-ссылки и контакты ботов не сохраняются.
- Не выполняется вход на сайты, обход авторизации или извлечение скрытых данных.
- Пользователь отвечает за законность обработки контактов и дальнейшей коммуникации.
- Не используйте результаты для спама; персонализируйте предложение и предоставляйте возможность отказаться от общения.

## История сообщений

- Бот записывает входящие текстовые сообщения, свои текстовые ответы и события отправки документов.
- Бинарное содержимое HTML, PDF и CSV не сохраняется в истории.
- Хранится до 200 последних записей пользователя в одном чате, но не дольше 90 дней.
- `/history` доступна для выгрузки только в личном чате, чтобы не раскрывать историю в группе.
- `/clear_history` удаляет записи пользователя в текущем чате; подтверждение удаления повторно не записывается.
- Другой пользователь не может получить историю по вашему `user_id`.

## Быстрая ручная проверка

1. Отправьте `/start` и проверьте наличие двух кнопок.
2. Нажмите **🎯 Лидогенерация**.
3. Ответьте на все пять вопросов.
4. На шаге «Где искать» укажите один или два известных публичных сайта.
5. Проверьте, что бот отправил `.csv` и показал меню с пакетным созданием презентаций.
6. Начните новую анкету и выполните `/cancel`.
7. Создайте тестовую презентацию по публичному сайту и откройте полученные HTML и PDF.
