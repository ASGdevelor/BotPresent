# BotPresent

Telegram-бот на Bun, TypeScript и [grammY](https://grammy.dev/), работающий локально без Docker:

- **📊 Презентации** — создаёт, обновляет и хранит персональные `index.html` и PDF по многоэкранному шаблону из `TestSite`, показывает прогресс 0–100% и наполняет страницы публичными данными конкретного сайта.
- **🎯 Лидогенерация** — ищет до 60 сайтов по точным и расширенным запросам, глубоко анализирует нишу, регион и внутренние страницы, ранжирует точные/частичные/похожие компании и выгружает CSV с владельцами, руководителями, менеджерами и контактами.

После лидогенерации кнопка **«Презентации для всех лидов»** последовательно создаёт HTML- и PDF-версии для каждого найденного сайта.

Проект работает без платных API и ключей AI. Для запуска нужен только токен Telegram-бота и исходящий доступ в интернет.

## Быстрый старт

### 1. Создайте Telegram-бота

1. Откройте [@BotFather](https://t.me/BotFather).
2. Выполните `/newbot` и следуйте инструкциям.
3. Сохраните полученный токен.

### 2. Настройте проект

Установите [Bun](https://bun.sh/), затем выполните:

```bash
git clone https://github.com/ASGdevelor/BotPresent.git
cd BotPresent
bun install
```

Скопируйте `.env.example` в `.env` и укажите токен:

```env
BOT_TOKEN=ваш_токен_от_BotFather
```

Для совместимости также распознаётся переменная `TOKEN`, но в новых конфигурациях рекомендуется `BOT_TOKEN`.

### 3. Запустите

```bash
bun run start
```

Режим разработки с автоматическим перезапуском:

```bash
bun run dev
```

## Команды

| Команда | Назначение |
| --- | --- |
| `/start` | Запустить бота и показать главное меню |
| `/menu` | Повторно показать главное меню |
| `/cancel` | Отменить текущее действие |
| `/history` | Выгрузить последние 100 сообщений в `.md` |
| `/clear_history` | Полностью удалить свою историю в текущем чате |

## Как работает бот

Состояние выбранного действия хранится в памяти процесса. Для лидогенерации бот последовательно запрашивает: кому продавать, кого искать, где искать, что предложить и кого исключить. После заполнения анкеты бот анализирует до 60 публичных сайтов и формирует CSV-отчёт в памяти. Точные совпадения выводятся первыми, затем частичные и похожие компании.

Для контакта указываются предполагаемая роль, подпись на сайте и страница-источник. Роль определяется эвристически по открытому тексту и требует ручной проверки. Приватные сетевые адреса, invite-ссылки и аккаунты ботов исключаются.

Входящие сообщения, ответы и события документов сохраняются в локальной SQLite не более 90 дней и 200 записей на пользователя в чате. Пользователь может выгрузить или удалить свою историю. База данных и её служебные файлы исключены из Git.

Презентации сохраняются в `data/presentations/`. Бот отправляет автономный редактируемый `index.html` и, если в системе найден Microsoft Edge или Google Chrome, PDF на одном непрерывном листе. PDF создаётся в фиксированной десктопной области 1280 px, поэтому на любом устройстве сохраняет размеры, отступы и взаимное положение элементов HTML; PDF-просмотрщик только масштабирует целый лист под экран. Сначала бот извлекает продуктовые направления, изображения и фирменную палитру, затем через Bing, DuckDuckGo и Google News исследует рынок, аудиторию и конкурентное поле. Источники сохраняются во внутренних данных для проверки, но презентация показывает не список ссылок, а продающий бизнес-разбор: цепляющий заголовок, сильные и слабые стороны бренда, сегменты аудитории, конкурентов, предложение AI-блогеров и 90-дневный план запуска. В HTML столбиковые и круговые диаграммы работают на встроенном Chart.js 4.5.1 без CDN; PDF печатает ту же экранную версию после полной загрузки диаграмм, шрифтов и изображений. Логотипы, продуктовые изображения и тематические GIF AI-блогеров встраиваются как `data:` URI.

После ввода сайта бот спрашивает, продаются ли компании AI-блогеры. Для ответа «да» автоматически выбирается отдельный `Generic/index3.html`: персональный разбор, ниша покупателей, сравнение с живыми блогерами, экономика, три роли AI-персонажей, первая серия, блок запуска в СНГ, рыночные доказательства и CTA пилота. Под бренд заново формируются заголовок, диагностика, аудитория, конкуренты, темы и графики. При каждой генерации сервер создаёт три анимированных GIF-персонажа под направления компании и перекрашивает их в выбранную палитру; фото из `Generic/assets` не используются. В обычном режиме используется улучшенный `Generic/index.html`: AI-формулировки и GIF не выводятся, а продуктовые фото берутся с анализируемого сайта. Оба шаблона содержат семь интерактивных Chart.js-графиков и автономные SVG-fallback.
