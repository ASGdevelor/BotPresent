# BotPresent

Telegram-бот на Bun, TypeScript и [grammY](https://grammy.dev/), работающий локально без Docker:

- **📊 Презентации** — создаёт, обновляет и хранит персональные `index.html` и PDF по многоэкранному шаблону из `TestSite`, показывает прогресс 0–100% и наполняет страницы публичными данными конкретного сайта.
- **🎯 Лидогенерация** — ищет до 60 сайтов по точным и расширенным запросам, глубоко анализирует нишу, регион и внутренние страницы, ранжирует точные/частичные/похожие компании и выгружает CSV с владельцами, руководителями, менеджерами и контактами.

После лидогенерации кнопка **«Презентации для всех лидов»** последовательно создаёт HTML- и PDF-версии для каждого найденного сайта.

Проект работает без платных API и ключей AI. Для запуска нужен только токен Telegram-бота и исходящий доступ в интернет.

## Быстрый старт

### 1. Создайте Telegram-бота

1. Откройте [@BotFather](https://t.me/BotFather).
2. Выполните `/newbot` и следуйте инструкциям.
3. Сохраните полученный токен.

### 2. Настройте проект

Установите [Bun](https://bun.sh/), затем выполните:

```bash
git clone https://github.com/ASGdevelor/BotPresent.git
cd BotPresent
bun install
```

Скопируйте `.env.example` в `.env` и укажите токен:

```env
BOT_TOKEN=ваш_токен_от_BotFather
```

Для совместимости также распознаётся переменная `TOKEN`, но в новых конфигурациях рекомендуется `BOT_TOKEN`.

### 3. Запустите

```bash
bun run start
```

Режим разработки с автоматическим перезапуском:

```bash
bun run dev
```

## Команды

| Команда | Назначение |
| --- | --- |
| `/start` | Запустить бота и показать главное меню |
| `/menu` | Повторно показать главное меню |
| `/cancel` | Отменить ожидание темы |
| `/history` | Выгрузить последние 100 сообщений в `.md` |
| `/clear_history` | Полностью удалить свою историю в текущем чате |

## Как работает бот

Состояние выбранного действия хранится в памяти процесса. Для лидогенерации бот последовательно запрашивает: кому продавать, кого искать, где искать, что предложить и кого исключить. После заполнения анкеты бот анализирует до десяти публичных сайтов и формирует отчёт `.md` в памяти. Компании с Telegram-контактами выводятся первыми, затем идут сайты без найденного контакта.

Для контакта указываются предполагаемая роль, подпись на сайте и страница-источник. Роль определяется эвристически по открытому тексту и требует ручной проверки. Приватные сетевые адреса, invite-ссылки и аккаунты ботов исключаются.

Входящие сообщения, ответы и события документов сохраняются в локальной SQLite не более 90 дней и 200 записей на пользователя в чате. Пользователь может выгрузить или удалить свою историю. База данных и её служебные файлы исключены из Git.

Текущая презентация представляет собой редактируемый базовый шаблон; фактическое наполнение следует адаптировать под задачу и аудиторию.
